Ubuntu 24.04LTS Minecraft Java Edition 1.20.4

Minecraft サーバー構築手順

Ubuntu上にMinecraft JEのマルチプレイサーバーを構築する手順です。

1
パッケージリストの更新とアップグレード

AWSインスタンスを起動後、SSH接続して最初に行うOSパッケージの更新作業です。

Terminal
sudo apt update
TIP
インストール可能なパッケージの最新リストを取得し、ローカルのデータベースを更新します。
sudo apt upgrade -y
TIP
パッケージを最新バージョンに更新します。-yオプションにより確認プロンプトを自動で承認します。
2
Java環境のインストールと作業ディレクトリ準備

Minecraftの動作に必要なJava実行環境（JDK）を導入し、専用フォルダを作成します。

Terminal
sudo apt install -y openjdk-17-jdk
TIP
Minecraft 1.20.4はJava 17環境が推奨のため、OpenJDK 17をインストールします。
cd /home/ubuntu
mkdir mc
TIP
/home/ubuntu/ にMinecraftサーバー用のディレクトリを作成します。
cd mc
wget https://piston-data.mojang.com/v1/objects/8dd1a28015f51b1803213892b50b7b4fc76e594d/server.jar
TIP
Minecraft公式から、バージョン1.20.4のサーバー実行ファイル（server.jar）を直接ダウンロードします。
3
サーバーの初回起動と各種設定

一度サーバーを起動して必要な設定ファイルを自動生成させ、利用規約や認証設定を変更します。

💡 メモリ設定に関するTips
Xmx（最大メモリ）とXms（最小メモリ）の割当量をどちらも同じ値に揃えることで、実行中の動的なメモリ再確保プロセスが抑えられ、サーバーのパフォーマンスが僅かに向上するとされています。
Terminal
java -Xmx3G -Xms3G -jar server.jar nogui
TIP
Xmx3G: 最大メモリ3GB / Xms3G: 最小メモリ3GB / nogui: GUI画面を表示せずCUI環境のみで起動。初回はeula.txt生成後に自動停止します。
① 利用規約への同意設定 (eula.txt)
nano eula.txt
eula=true
TIP
falseからtrueに変更することで規約に同意し、サーバーの起動が可能になります。
② サーバープロパティの編集 (server.properties)
nano server.properties
online-mode=false
TIP
今回は非正規クライアント（TLauncherなど）を利用するため、Microsoftアカウントによる認証を無効化します。
4
本起動と停止コマンド

設定が完了したら再度起動を行います。

Done (10.818s)! For help, type "help" が出れば起動成功です。

Minecraftクライアントを持っている場合は、タイトル→マルチプレイ→サーバーを追加 から接続してみましょう。

サーバー名：任意
サーバーアドレス：AWSのパブリックIP

Terminal
java -Xmx3G -Xms3G -jar server.jar nogui
stop
TIP
Minecraftサーバー内の終了コマンドです。ワールドデータを保存して安全に停止します。

stopコマンドを実行し、Linuxのプロンプトに戻ることを確認してください。
