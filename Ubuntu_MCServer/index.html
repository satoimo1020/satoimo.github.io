1
パッケージリストの更新とアップグレード

AWSインスタンスを起動後、SSH接続して最初に行うOSパッケージの更新作業

Terminal
sudo apt update
TIP
インストール可能なパッケージの最新リストを取得し、ローカルのデータベースを更新
sudo apt upgrade -y
TIP
パッケージを最新バージョンに更新。-yオプションにより確認プロンプトを自動で承認
2
Java環境のインストールと作業ディレクトリ準備

Minecraftの動作に必要なJava実行環境（JDK）を導入し、専用フォルダを作成する

Terminal
sudo apt install -y openjdk-17-jdk
TIP
Minecraft 1.20.4はJava 17環境が推奨のため、OpenJDK 17をインストール
java -version
TIP
正常にインストールできているかの確認
結果：openjdk version "17.X.XX" 2026-XX-XX
cd /home/ubuntu
mkdir mc
TIP
/home/ubuntu/ にMinecraftサーバー用のディレクトリを作成
cd mc
wget https://piston-data.mojang.com/v1/objects/8dd1a28015f51b1803213892b50b7b4fc76e594d/server.jar
TIP
Minecraft公式から、バージョン1.20.4のサーバー実行ファイル（server.jar）を直接ダウンロード
ls -lh server.jar
TIP
権限の確認
結果：-rw-rw-r-- 1 ubuntu ubuntu 47M Dec 7 2023 server.jar
3
サーバーの初回起動と各種設定

一度サーバーを起動して必要な設定ファイルを自動生成させ、利用規約や認証設定を変更する

💡 メモリ設定に関するTips
Xmx（最大メモリ）とXms（最小メモリ）の割当量をどちらも同じ値に揃えることで、実行中の動的なメモリ再確保プロセスが抑えられ、サーバーのパフォーマンスが僅かに向上するらしい
Terminal
java -Xmx3G -Xms3G -jar server.jar nogui
TIP
Xmx3G: 最大メモリ3GB / Xms3G: 最小メモリ3GB / nogui: GUI画面を表示せずCUI環境のみで起動。初回はeula.txt生成後に自動停止する
① 利用規約への同意設定 (eula.txt)
nano eula.txt
eula=true
TIP
falseからtrueに変更することで規約に同意し、サーバーの起動が可能になる
② サーバープロパティの編集 (server.properties)
nano server.properties
online-mode=false
TIP
今回は非正規クライアント（TLauncher）を利用するため、Microsoftアカウントによる認証を無効化
4
本起動と停止コマンド

設定が完了したら再度起動を行う

Done (10.818s)! For help, type "help" が出れば起動成功

Minecraftクライアントを持っている場合は、タイトル→マルチプレイ→サーバーを追加 から接続してみる

サーバー名：任意
サーバーアドレス：AWSのパブリックIP

Terminal
java -Xmx3G -Xms3G -jar server.jar nogui
stop
TIP
Minecraftサーバー内の終了コマンド。ワールドデータを保存して安全に停止できる

stopコマンドを実行し、Linuxのプロンプトに戻ることを確認

5
Minecraftサーバーのサービス化

systemdに登録し、バックグラウンドでの常時起動や自動管理を行えるようにする

① サービス定義ファイルの新規作成
sudo nano /etc/systemd/system/minecraft.service
[Unit] Description=Minecraft server After=network-online.target [Service] Type=forking User=ubuntu Group=ubuntu WorkingDirectory=/home/ubuntu/mc ExecStart=screen -S minecraft -d -m java -Xms3G -Xmx3G -jar /home/ubuntu/mc/server.jar nogui ExecStop=screen -S minecraft -X quit Restart=on-failure [Install] WantedBy = multi-user.target
Terminal
sudo systemctl daemon-reload
TIP
systemdのユニットファイルを再読込し、作成した.serviceファイルをシステムに認識させる
6
Minecraftサーバーの自動実行

OSが再起動した際などに、Minecraftサーバーが自動的に立ち上がるよう設定する

Terminal
sudo systemctl enable minecraft
TIP
サービスを有効化し、システム起動時に自動実行されるように登録する
sudo systemctl is-enabled minecraft
TIP
自動起動設定が正しく有効になっているかを確認（enabledと返ってくればOK）
7
screenによるセッション管理

サービス経由で仮想端末（screen）上で動くサーバーのコンソールを確認・操作する方法

Terminal
screen -ls
TIP
現在バックグラウンドで起動しているscreenセッションの一覧を表示
screen -r minecraft
TIP
「minecraft」セッションにアタッチし、サーバーの生ログ確認やコマンド入力画面に入る
💡 デタッチ（作業画面から安全に抜ける）
Ctrl + A キーを押した後に D キーを押す

※この操作を行うことで、サーバーを起動したまま元のLinuxプロンプトに戻ることができる
