test1 (Zabbixサーバー) の構築

test1（Zabbixサーバー）に、Zabbixサーバー、Webフロントエンド、データベース（MariaDB）を構築する。

1-1
Zabbixリポジトリの登録

Zabbix公式のリポジトリをシステムに追加する。wgetで取得するファイルはインストール後、不要になるため tmp ディレクトリにダウンロードする。

Terminal
cd /tmp
wget https://repo.zabbix.com/zabbix/7.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_7.0-1+ubuntu24.04_all.deb
TIP
Ubuntu 24.04用のZabbix 7.0公式リポジトリパッケージをダウンロード
sudo dpkg -i zabbix-release_7.0-1+ubuntu24.04_all.deb
TIP
ダウンロードしたリポジトリパッケージをシステムにインストール
sudo apt update
TIP
新しく追加されたZabbixリポジトリのパッケージリストを同期
1-2
必要なパッケージをインストール

Zabbixサーバー、フロントエンド、Apache、およびZabbixエージェントを一括でインストール

⚠️ DBに関する注意事項
Ubuntu すでに新しいバージョンのMariaDBが存在する状態で、それよりも古いバージョンのMariaDBを上書きインストールしようとすると、データの互換性がなくなってシステムがフリーズするみたい。 新たにインストールしないように。（今回は既にインストール済みのため除外）
Terminal
sudo apt install -y zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-sql-scripts zabbix-agent
TIP
Zabbixに必要な各種コンポーネントをインストール（-yで自動承認）
1-3
データベースの初期設定

MariaDBに接続し、Zabbix用のデータベースとユーザーを作成する。

Terminal
sudo mariadb
TIP
MariaDBにroot権限で接続（パスワードレス、または設定したrootパスワードを入力）
MariaDB プロンプト内でのSQL実行

※ 'password' 部分で zabbix ユーザーのパスワードを設定している。

create database zabbix character set utf8mb4 collate utf8mb4_bin; create user zabbix@localhost identified by 'password'; grant all privileges on zabbix.* to zabbix@localhost; set global log_bin_trust_function_creators = 1; quit;
1-4
初期データのインポート

Zabbixが提供する初期スキーマとデータを、先ほど作成したデータベースに流し込む。

Terminal
zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql --default-character-set=utf8mb4 -uzabbix -p zabbix
TIP
パスワードを求められるため、[1-3]で設定した zabbix ユーザーのパスワードを入力。数分程かかる場合がある。

インポートが終わったら、一時設定を元に戻す。

Terminal
sudo mariadb
MariaDB プロンプト内でのSQL実行
set global log_bin_trust_function_creators = 0; quit;
1-5
日本語化

日本語パッケージをインストール

Terminal
sudo apt install -y language-pack-ja-base language-pack-ja

日本語を使用するように設定

Terminal
echo 'export LANG=ja_JP.UTF-8' >> ~/.bashrc
echo 'export LANGUAGE="ja_JP:ja"' >> ~/.bashrc
source ~/.bashrc

日本語ロケールを有効化

① ロケール設定ファイルの編集
sudo nano /etc/locale.gen

ファイル内で以下の行を探して、コメントアウト（#）を解除

ja_JP.UTF-8 UTF-8
1-6
Zabbixサーバーの設定編集

Zabbixサーバーの構成ファイルを編集し、データベースのパスワードを指定する。

① 構成ファイルの編集
sudo nano /etc/zabbix/zabbix_server.conf

ファイル内で `DBPassword=` の行を探して、コメントアウト（#）を外し、データベースのパスワードを指定

DBPassword=password
TIP
[1-3]で設定した zabbix ユーザーのパスワードを記述
1-7
サービスの起動と有効化

各種サービスを起動し、OS起動時に自動実行されるよう設定

Terminal
sudo systemctl restart zabbix-server zabbix-agent apache2 mysql
sudo systemctl enable zabbix-server zabbix-agent apache2 mysql
sudo systemctl is-active zabbix-server zabbix-agent apache2 mysql
TIP
起動しているかを確認（activeになっていればOK）

ports.conf を編集

sudo nano /etc/apache2/ports.conf

ポートを 8080 から 80 に変更

Listen 8080 → Listen 80
TIP
こっちのほうが都合がいい

使用ポートの変更を適用するために再起動

sudo systemctl restart apache2
1-8
WebUIでの初期セットアップ

ホストOSのブラウザから、http://192.168.56.201/zabbix にアクセスする。

💡 セットアップウィザードの手順
  1. ようこそ: 言語を選択（日本語が選べるようになっているか確認）して「次のステップ」
  2. 前提条件のチェック: 全て「OK」であることを確認して「次のステップ」
  3. データベース接続設定: Password 欄に [1-3] で決めたパスワードを入力して「次のステップ」（※他の設定は触らない）
  4. 設定: サーバー名（今回は"test1"）を入力して「次のステップ」
  5. インストール: 設定内容を確認して「終了」
🔐 初期ログイン情報
  • ユーザー名: Admin (※イニシャルが大文字になっているので注意)
  • パスワード: zabbix
1-9
グラフの日本語化

Zabbix のグラフで使用するフォントをインストール

Terminal
sudo apt install -y fonts-takao-pgothic

インストールしたフォントを有効化

Terminal
sudo update-alternatives --install /usr/share/zabbix/assets/fonts/graphfont.ttf zabbix-frontend-font /usr/share/fonts/truetype/takao-gothic/TakaoPGothic.ttf 20

フォントを適用するため、各種サービスを再起動

Terminal
sudo systemctl restart zabbix-server zabbix-agent apache2
