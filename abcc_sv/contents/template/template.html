1. ターミナル表示とプロンプトのバリエーション

各種CUI（コマンドライン環境）の操作手順を区別して記述するための、複数のプロンプト形式と、補足用のホバーポップアップ（TIP）の見本です。

1-1
一般ユーザープロンプト（$）とTIP

一般権限で実行するコマンドには data-prompt="$" を使用します。また、コマンドラインの右端に tip-indicator を配置することで、ホバーで表示される補足（TIP）を埋め込めます。

Terminal (一般ユーザー権限)
cd /path/to/workspace
curl -O https://example.com/sample-package.tar.gz
TIP
外部サーバーからサンプルパッケージをカレントディレクトリにダウンロードします
tar -zxvf sample-package.tar.gz
TIP
ダウンロードした圧縮ファイルをアーカイブ展開します
1-2
ルートユーザープロンプト（#）と注意書き

システム管理者権限（root）を明示する場合は、data-prompt="#" を指定します。また、重要な警告や注意を促す場合は note-box コンポーネントを配置します。

⚠️ 実行前の注意事項
以降のステップはシステムに影響を与える設定変更を含みます。必ずバックアップを取得した上で、慎重にコマンドを入力してください。誤った設定を行うとOSが起動しなくなる恐れがあります。
Terminal (Root 権限)
sudo -i
apt update && apt upgrade -y
TIP
システムリポジトリの同期と全パッケージの一括アップデート（自動承認）
systemctl stop legacy-service
1-3
インタラクティブプロンプト（>）

データベースのシェル（MySQL、MariaDB等）や、特定の対話型コンソール内部での入力を表現する場合は data-prompt=">" を用います。

Terminal (MySQL / コンソール接続)
mysql -u root -p
対話型シェル（プロンプト内）でのコマンド実行
Database Prompt
CREATE DATABASE demo_db CHARACTER SET utf8mb4;
CREATE USER 'demo_user'@'localhost' IDENTIFIED BY 'password';
FLUSH PRIVILEGES;
exit

2. ファイル編集と外部リンク

設定ファイルなどのソースコード、追記すべき箇所、およびGUI操作への動線（外部リンク・クリック可能なURL）の記述方法の見本です。

2-1
ファイル内容（file-content）とハイライト

テキストエディタ（nano, vim等）でファイルを開く手順と、その中に記述するコード内容、および編集すべきポイントを highlight タグで囲んで強調表示させます。

エディタによるファイル編集コマンド
sudo nano /etc/example/config.conf

編集対象ファイル内の記述内容（該当の箇所を見つけてハイライトのように書き換えます）

/etc/example/config.conf
# 基本ネットワーク設定
NetworkAddress=192.168.1.1
EnableServicePort=8080
DebugMode=true
TIP
外部から接続可能にするため、標準ポートを8080に変更し、デバッグモードを有効にします
2-2
クリック可能なURLリンクと手順リスト

Webブラウザから管理画面や外部の参考サイトへ移動させる際、視認しやすいクリック可能なURLリンク（アンカータグ）を設置します。

👉 対象WebUIのアクセスURL: https://example.com/dashboard

💡 ブラウザ側のセットアップ操作手順
  1. 上記リンクから、管理コンソールのログイン画面にアクセスします。
  2. 初期設定の認証: ログインフォームに、下記注意ボックスの初期認証情報を入力して「サインイン」を押します。
  3. プロファイル設定: 基本言語に「日本語」を選択して次へ進みます。
  4. 初期構成の完了: 画面のガイダンスに従ってセットアップウィザードを「完了」させます。
🔐 テスト環境用の初期アカウント情報
  • Administrator ID: admin_user (※末尾のアンダーバーに注意)
  • Default Password: TempPass1234

3. データ比較・マトリックス

複数の環境や、サーバー/クライアント間で異なるパラメーターを設定させる場合、表形式（マトリックス）を用いることで見やすく比較できます。

3-1
複数環境の対比マトリックス（テーブル）

行ごとに設定項目を並べ、列ごとに異なる環境（例: 本番環境、テスト環境、開発環境など）に適用すべきパラメータを表現するマトリックス構成です。

4. 画像表示

複数の環境や、サーバー/クライアント間で異なるパラメーターを設定させる場合、表形式（マトリックス）を用いることで見やすく比較できます。

4-1
画像を表示する

画像を表示する際は、カレントの /images フォルダー内に保存された画像を使用する。

画像の命名規則は、ステップ番号_通し番号_補足説明（例: 4-1_01_sample.png）とする。
