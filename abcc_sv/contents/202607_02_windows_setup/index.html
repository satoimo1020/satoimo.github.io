1. VirtualBox側の設定

VirtualBox上でWindows 11を正常に動かすための仮想マシンの新規作成、およびインストールの事前準備となるリソース割当とハードウェア設定を行います。

1-1
仮想マシンの新規作成
新規マシン作成
  1. VirtualBoxを起動し、上部メニューの「新規」をクリック
  2. 仮想マシンの名前と OS:
    • VM名: 任意（例: Win11_24H2）
    • ISOイメージ: 事前にダウンロードしたWindows 11のisoファイルを選択
    • 無人インストールを実行: チェックを外す
  3. 仮想ハードウェアを指定:
    • メインメモリー: 8192MB 推奨
    • CPU数: 8コア 推奨
  4. 設定内容を確認し、「完了」
1-2
仮想ハードウェア・詳細設定

作成した仮想マシンの設定を最適化し、ローカルアカウント作成をスムーズに行うためにネットワークを遮断します。

仮想マシンの設定変更
  1. 作成した仮想マシンを選択した状態で「設定」を開く
  2. ディスプレイ ＞ 「グラフィックスコントローラー」を VBoxVGA に変更
    TIP
    VBoxSVGAを利用すると起動できない事象を確認したため、VBoxVGAを選択する
  3. ストレージ ＞ VM名.vdi を選択 ＞ 「SSD（ソリッドステートドライブ）」にチェックを入れる
  4. ネットワーク ＞ 「ネットワークアダプターを有効化」のチェックを外す
  5. 「OK」をクリックして変更を保存

3. OOBEネットワーク制限の回避

Windows 11のセットアップでは、ネットワーク接続とMicrosoftアカウントの紐付けをスキップできない。 ローカルアカウントでセットアップを完了させるために、制限を回避するコマンドを投入する。

参考: OOBEとは - IT用語辞典 e-Words

3-1
レジストリ編集

初期インストール完了後、国・地域選択などの「OOBE画面」が起動したら以下の作業を実施する。

cmdを起動

最初の選択画面（国選択等）が表示された状態で、キーボードの Shift + F10 キーでコマンドプロンプトを立ち上げる

cmd
reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\OOBE /v BypassNRO /t REG_DWORD /d 1 /f
shutdown -r -t 0 -f
TIP
shutdown: シャットダウンコマンド
-r: 再起動
-t 0: 0秒後に実行
-f: 強制実行

※もしシャットダウン状態のままフリーズした場合は、強制的に電源を落として再起動する

5. インストール完了後の設定

デスクトップ画面が表示された後、Windowsを使用可能な状態にする。

5-1
Windows Updateの一時停止
設定変更
  1. スタートメニュー等から「設定」を開く
  2. Windows Update タブ ＞ 更新の一時停止 ＞ 「1週間一時停止する」 に変更します。
⚠️ なぜ直後にWindows Updateを停止するのか？
インストール直後に通信を戻すと、バックグラウンドで Windows Update が始まってしまう。大量のアップデートデータをダウンロードすることになるため、今回は停止する。
5-2
VMの各種設定
VirtualBox設定
  1. VirtualBox側のNWアダプター有効化:
    • VMのWindowsをシャットダウン
    • 当該マシンの 設定 ＞ ネットワーク ＞ 「ネットワークアダプターを有効化」のチェックをつける
  2. 通信回復の確認:
    • 再度仮想マシンを起動させ、Windowsがインターネットへ疎通していることを確認

最後に、Windowsをシャットダウンしてスナップショットを取る。
